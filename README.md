## How the GitHub commits counter works

The hero card shows **"N commits in the last 365 days."** That number is not
fetched live from a third-party service. Instead:

1. `.github/workflows/update-stats.yml` runs daily at 06:30 UTC (and on
   manual dispatch).
2. It calls the GitHub GraphQL API for
   `user.contributionsCollection.totalCommitContributions` on a rolling
   365-day window, using the default `GITHUB_TOKEN`.
3. It writes the result to `stats.json` and pushes a commit with
   `[skip ci]` if (and only if) the numbers changed.
4. `assets/js/main.js` fetches `stats.json` on page load and fills in the
   count. If the fetch fails the UI falls back gracefully.

This approach is deliberately simple: no external service, no API key to
rotate, and the value lives in the repo as committed history тАФ so it's
inspectable and versioned.

### Forcing a refresh

- GitHub тЖТ this repo тЖТ **Actions** тЖТ **Update GitHub stats** тЖТ **Run workflow**.
- Or push any change to `.github/workflows/update-stats.yml`.

## Deploying changes

GitHub Pages auto-deploys on push to `main`. Typical workflow:

```bash
# edit index.html / style.css / etc.
git add .
git commit -m "..."
git push
```

Then wait ~30тАУ60 s for the Pages deploy. The domain
`christopherfrancisque.com` points at the same site.

## Editing content

The site is intentionally a single `index.html` file. Content lives in
plain markup тАФ no templating тАФ so you edit where the content appears:

- **Current role / intro**: the `<section class="hero">` block.
- **Experience**: `<ol class="timeline">` inside `<section id="experience">`.
  Each role is a `<li class="timeline__item">`. Order is top-to-bottom =
  most recent first.
- **Projects**: `<article class="project">` cards inside
  `<section id="projects">`.
- **Skills**: `<ul class="chips">` inside `<section id="skills">`.
- **Resume PDF**: drop a replacement at `assets/files/resume.pdf`. The
  filename is referenced in two places (hero button + `section#resume` CTA).

```bash
git commit -m "chore(resume): update to <month> <year>"

```

## Local preview

No build step. Any static server works:

```bash
# Python 3
python3 -m http.server 8080
# or Node
npx http-server -p 8080 .
```

Then open [http://localhost:8080](http://localhost:8080).

## Accessibility + performance notes

- Semantic landmarks (`<header>`, `<main>`, `<section>`, `<footer>`).
- Keyboard-reachable skip link.
- `prefers-reduced-motion` respected (pulse + scroll-reveal disabled).
- Headshot optimized to ~45 KB (from a 3.7 MB original).
- Fonts loaded via Google Fonts; no blocking CSS beyond the stylesheet.
- `main.js` is `defer`-loaded, no render-blocking JS.
